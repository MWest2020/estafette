## Context

Reports already exist as deterministic `report.json` files under `reports/`,
each validated by the `TransferabilityReport` model. The catalogue is a pure
view over them. The design goal is the smallest honest platform: a static site,
generated by the CLI, hostable on GitHub Pages — no server or DB to run or
secure.

## Goals / Non-Goals

**Goals:**

- `estafette catalogue` renders `reports/*/report.json` to a static site.
- Deterministic output (I5); self-contained HTML + inline CSS.
- Deploy to GitHub Pages via a workflow.

**Non-Goals:**

- No server, no database engine, no web framework, no JS build step.
- No search/filter/auth (later, if ever).
- No change to the verdict path — the site only renders what the engine decided.

## Decisions

- **`reports/` is the datastore.** The catalogue re-uses the `TransferabilityReport`
  model (`model_validate_json`) to load each report — no second schema. A real
  database can slot in later behind the same loader without touching the renderer.
- **Static HTML, inline CSS, zero JS framework.** Matches invariant I2 (don't
  reinvent) and keeps hosting trivial. A tiny amount of inline CSS for a readable
  table + badges; no external assets so it works offline and on Pages.
- **Deterministic by construction.** Reports are sorted by (name, commit); no
  timestamps or absolute paths are emitted. A determinism test asserts
  byte-identical repeats, mirroring the report body guarantee.
- **One entry per report file (per commit).** MVP lists every report found.
  Grouping "latest per PoC" is a later refinement, not needed to show PoCs.
- **Pages via workflow.** `pages.yml` runs `estafette catalogue --out site` and
  deploys with the official Pages actions. Enabling Pages (source = GitHub
  Actions) is a one-time repo setting Mark flips.
- **CLI-emitted artifact, not a service.** "estafette is a CLI" still holds; the
  platform is generated output, preserving the safety posture (nothing dynamic).

## Risks / Trade-offs

- **Report volume over time.** Append-only reports accumulate; the index could
  grow long. Accepted for MVP; grouping/pagination is a later concern.
- **HTML escaping.** Gap messages and names are injected into HTML and MUST be
  escaped to avoid broken markup (and defensively, injection). Use stdlib
  `html.escape`; a test covers a name with markup characters.
- **Schema drift.** If the report schema changes, old `report.json` files must
  still load. Mitigation: load leniently and skip/annotate unparseable reports
  rather than crashing the whole build.
- **Pages enablement is manual.** The first deploy needs the repo's Pages source
  set to "GitHub Actions"; documented, not automatable from here.
